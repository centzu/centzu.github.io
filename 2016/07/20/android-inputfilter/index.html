以前在做一个安全卫士的android Demo的时候，要用到EditText的输入控制，于是就顺手去baidu（google）TextWatcher和InputFilter了，Fuck，恕我直言，那些排在前面的结果基本都是辣鸡，直接贴了一大串代码，没解析，甚至有几个博客内容一模一样，真是的！

最后用InputFilter做了一个简单的工具类来控制EditText的输入

了解InputFilter

public interface InputFilter
InputFilters can be attached to Editables to constrain the changes that can be made to them.

InputFilter是一个接口,它可以绑定在编辑框上限制它们的输入。

它有一个抽象方法filter；

This method is called when the buffer is going to replace the range dstart … dend of dest with the new text from the range start … end of source.

不知道是不是我的理解有问题，上述谷歌API的解释好像说用从start-end长度的source来替代dstart-dend长度的dest，~那么source应该就是输入后整个文本，dest是输入前的整个文本~
>
我后面测试了一下，发现不是这样的

  • 假如原文本是:abcde
  • 然后我输入了一个w
  • 打印出来的结果是:
    source : w
    dest : abcde

多次测试也是同样的结果，如果有错请指正。
于是就有这样解析

Return the CharSequence that you would like to have placed there instead, including an empty string if appropriate, or null to accept the original replacement.

方法最后会返回一个你想要的字符串来进行替代（就是把返回的字符串加入原文本后面），一般输入正确返回source，错误返回“”，这是我的理解。
